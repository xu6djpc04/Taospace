## 絕對紅線（Claude Code 快速參照）

> 以下紅線為 Claude Code 快速參照，完整規則見自動載入的 `AGENTS.md`。

1. **`Taofiles/` 唯讀**：絕對不修改、不刪除、不覆蓋原始檔。
2. **產出永遠另存新檔於 `output/`**，不覆蓋原始檔。覆蓋 `output/` 內非本次任務所建的既有檔前，**先告知人、由人決定**。
3. **不得在未經人確認下自行改寫 `AGENTS.md`、`CLAUDE.md`、`methodology.md` 等規則檔**。
4. **有後果的動作先確認**：批次產出、大幅改寫、改規則前，先說明範圍、等人確認，不自行決定動手。以下**一律停下確認**（不靠判斷）：**刪除任何檔案**（含 `output/`、`references/`）、**覆蓋既有檔案**、**單次批次 > 10 個檔案**、**修改任何規則檔**。
5. **確認問句一旦出現，當輪即停**：確認問句與執行動作不得出現在同一個 response 中；問句送出後，須等待人的明確回覆才能繼續。**嚴格定義「明確回覆」**：僅當使用者給出「好」、「同意」、「執行」等無歧義的許可時，才視為通過；回覆語意模糊（如「幫我規劃」、「你看著辦」）或偏離原確認問題，一律視為「未授權」，必須再次提問釐清，絕不可自行腦補預設為同意並動手。
6. **不捏造**：人名、日期、職稱直接轉錄；不確定標 `（待確認）`，不推測填補。
7. **日期一律用民國年數字**（不寫西元、不加「民國」字樣）：含完整日期用 `YYYMMDD` 七碼（如 `1150608`）。

## Claude Code 專屬：讀取文件優先用 Skill

遇到以下檔案類型，Claude Code **讀取**時優先使用對應 skill，不要自行解析：

| 檔案類型 | skill | 呼叫方式 |
|----------|-------|---------|
| `.docx` / `.doc` | docx skill | 「用 docx skill 讀取 ...」 |
| `.xlsx` / `.xls` | xlsx skill | 「用 xlsx skill 讀取 ...」 |
| `.pdf` | pdf skill | 「用 pdf skill 讀取 ...」 |
| `.pptx` / `.ppt` | pptx skill | 「用 pptx skill 讀取 ...」 |

Skills 來源：`anthropic-agent-skills` 套件（`document-skills`），已安裝於此專案。
原始碼備份：`C:\Users\kevin\.claude\skills\`

**建立新文件**一律用 Python 套件（`python-docx` / `openpyxl` / `python-pptx`），與其他工具相同——見 `AGENTS.md` 第 6 節。
