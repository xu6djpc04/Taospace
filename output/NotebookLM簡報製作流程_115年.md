# NotebookLM → 簡報 製作流程 SOP

> 版本：v1.0　建立：1150618
> 適用：把 NotebookLM 記事本內的多來源資料，整理成可用的簡報。
> 核心原則：**美化交給工具，內容正確性由人＋Claude Code 把關。**

---

## 流程總覽

```
① NotebookLM 彙整來源
        ↓（MCP 讀取，引文逐字轉錄）
② Claude Code 整理 MD 設計稿（含來源快照）
        ↓（貼進 claude.ai/design）
③ Claude Design 產出 PPT
        ↓（pptx skill 讀回核對）
④ Claude Code 校對閉環 → 確認無誤
```

| 步驟 | 工具 | 產出 | 把關重點 |
|------|------|------|---------|
| ① | NotebookLM | 已索引的來源 | 來源齊全、確認哪些要納入 |
| ② | Claude Code | `output/{主題}設計稿_{年度}.md` | 引文準確、轉繁、命名規範 |
| ③ | Claude Design | `.pptx` | 排版、配色、視覺 |
| ④ | Claude Code | 校對報告 | 引文、人名、章節無錯字 |

---

## 各步驟做法與注意事項

### ① NotebookLM 彙整來源
- 確認**這次要納入哪些來源**（來源會增刪，先講清楚）。
- 大型 PDF（超過讀取上限）先在 NotebookLM 內用 query／note 分章摘要，不要硬讀全文。

### ② Claude Code 整理 MD 設計稿
- **只產 MD，不產 pptx**（排版是 Design 的事，CC 排版會白做）。
- **引文用「轉錄」不用「重述」**：經文、人名、章節直接從來源原文複製，不要憑記憶重打 → 根治錯字。
- MD 開頭固定放**來源快照**：
  ```
  > 依據來源（讀取日 115xxxx）：
  > 1. 檔名A
  > 2. 檔名B
  ```
  來源一變，一眼看出設計稿過期。
- 不確定的內容標 `（待確認）`，不推測填補。
- 命名：`{主題}設計稿_{年度}.md`，存 `output/`。

### ③ Claude Design 產出 PPT
- 把 MD 設計稿整份貼進 claude.ai/design。
- MD 裡先寫好配色、版式指令（如：深棕 #4A2C00 / 金黃 #C8A000 / 米白 #FDF6E3）。
- 成果 pptx 建議存 `output/`（勿存 `Taofiles/`，那是唯讀原始區）。

### ④ Claude Code 校對閉環（最重要、別省）
- 用 pptx skill 把成果 PPT **讀回來**。
- 逐句把 PPT 上的引文、人名、章節**對回 NotebookLM 原始來源**。
- 列出可疑處交人確認；確認後才算完成。
- 理由：Design 會照著錯字做圖，美化會掩蓋內容錯誤。

---

## 已知風險

- **引文錯字**：重打容易錯（已實際發生），務必轉錄＋校對。
- **來源變動**：刪／增來源後，設計稿與 PPT 都需重核，靠「來源快照」判斷是否過期。
- **大檔遺漏**：超上限的 PDF 若沒摘要納入，內容會缺，需明確標注是否已納入。
